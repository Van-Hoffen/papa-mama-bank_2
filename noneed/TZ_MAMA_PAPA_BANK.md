# 📋 ТЕХНИЧЕСКОЕ ЗАДАНИЕ (ТЗ)

## Веб-приложение "Мама-Папа Банк"
### Платформа для обучения детей управлению финансами

---

## 1. ОПИСАНИЕ ПРОЕКТА

**Название:** Мама-Папа Банк  
**Тип:** Веб-приложение (SPA + Backend API)  
**Назначение:** Образовательная платформа для обучения детей финансовой грамотности через систему виртуальных вкладов с разными условиями.

**Целевая аудитория:**
- Дети (7-18 лет) - пользователи-вкладчики
- Родители (25+ лет) - администраторы системы

**Основная идея:** 
Система состоит из двух "банков" (Мама-банк и Папа-банк) с разными условиями хранения денег. Ребёнок выбирает банк, открывает вклад, и система автоматически начисляет проценты. Все операции требуют одобрения от родителя через систему заявок.

## 2. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### 2.1 Система авторизации и ролей

**Требование 2.1.1:** Регистрация пользователей
- Поля: Имя, Логин, Пароль, Роль (Администратор/Вкладчик)
- Валидация: логин не менее 4 символов, пароль не менее 6 символов
- Хеширование паролей: bcryptjs (10 раундов)
- Запрет на дублирование логинов

**Требование 2.1.2:** Вход в систему
- Email/логин + пароль
- JWT токен (7 дней действия)
- Автоматическое сохранение сессии в localStorage (frontend)
- Логаут с удалением токена

**Требование 2.1.3:** Двухролевая система
- **Администратор (Родитель):** управление операциями, одобрение заявок, просмотр статистики
- **Вкладчик (Ребёнок):** создание вкладов, запрос операций, просмотр балансов

### 2.2 Система вкладов

**Требование 2.2.1:** Два типа банков с жёсткими условиями

| Параметр | Мама-банк | Папа-банк |
|----------|-----------|-----------|
| **Процентная ставка** | 3.5% за 14 дней | 11% за 30 дней |
| **Минимальный вклад** | 1000 ₽ | 2000 ₽ |
| **Штраф за досрочное снятие** | 0% (нет) | -2% |
| **Капитализация** | Каждые 14 дней | Каждые 30 дней |

**Требование 2.2.2:** Создание вклада
- Ребёнок заполняет форму: выбор банка, сумма
- Отправляет заявку на одобрение родителю
- Статус: Pending → Active (после одобрения) / Rejected
- Вклад хранит: сумму, текущий баланс, дату создания, статус

**Требование 2.2.3:** Управление вкладами
- Просмотр всех вкладов ребёнка
- Отображение: начальная сумма, текущий баланс, заработано процентов
- Фильтрация по статусу (Active, Closed, Pending)
- История по каждому вкладу

### 2.3 Система расчёта процентов

**Требование 2.3.1:** Автоматический расчёт
- Формула сложного процента: A = P × (1 + r)^n
  - P = начальная сумма
  - r = процентная ставка (3.5% или 11%)
  - n = количество завершённых периодов
- Расчёт выполняется на лету при каждом запросе (динамический)

**Требование 2.3.2:** Логирование процентов
- Сохранение истории каждого начисления в таблице interest_log
- Запись: дата, начисленная сумма, новый баланс

**Требование 2.3.3:** Прогнозы доходности
- Расчёт прогноза на: 7, 14, 30, 90 дней
- API endpoint: `/api/analytics/forecast/:depositId`

### 2.4 Система операций (заявки)

**Требование 2.4.1:** Типы операций
- `open` — открыть новый вклад
- `withdraw` — снять деньги
- `interest` — начисление процентов (система)
- `penalty` — штраф (для Папа-банка)

**Требование 2.4.2:** Жизненный цикл операции
1. Ребёнок создаёт заявку (статус: `pending`)
2. Родитель видит заявку в панели
3. Родитель одобряет (`approved`) или отклоняет (`rejected`)
4. При одобрении операция выполняется

**Требование 2.4.3:** Одобрение открытия вклада
- При одобрении заявки `open` статус вклада меняется на `active`
- Начинается начисление процентов

**Требование 2.4.4:** Одобрение снятия
- Проверка: прошло ли минимум 14 дней (Мама) или 30 дней (Папа)
- Если раньше → применение штрафа (-2% для Папа-банка)
- Создание записи о штрафе в operations
- Вклад закрывается

### 2.5 Панель администратора (Родитель)

**Требование 2.5.1:** Главная панель
- Список всех ожидающих операций
- Для каждой: имя ребёнка, тип операции, сумма, банк, дата запроса
- Кнопки: "Одобрить" и "Отклонить"

**Требование 2.5.2:** Статистика
- Общее количество детей в системе
- Общая сумма инвестирована
- Количество ожидающих операций
- Общие проценты, заработанные детьми

**Требование 2.5.3:** История операций
- Просмотр всех прошлых операций (все дети)
- Фильтрация по статусу, дате, типу

### 2.6 Личный кабинет ребёнка

**Требование 2.6.1:** Главная страница
- Сводная статистика:
  - Всего денег на счёте (сумма всех активных вкладов)
  - Всего инвестировано
  - Заработано процентов
  - Количество активных вкладов

**Требование 2.6.2:** Список вкладов
- Карточка для каждого вклада с информацией:
  - Банк (иконка Мама/Папа)
  - Начальная сумма
  - Текущий баланс
  - Заработано процентов
  - Статус (активен, закрыт, ожидание)
  - Метаданные: ставка, период, дни в вкладе

**Требование 2.6.3:** Прогноз доходности
- По клику на вклад показать прогноз на 7, 14, 30, 90 дней
- Примерное предупреждение для Папа-банка: "Осталось X дней до снятия без штрафа"

**Требование 2.6.4:** Создание вклада
- Модальное окно с формой:
  - Выбор банка (радиокнопки)
  - Сумма (input number)
  - Минимум и шаг указаны явно
  - Условия отображены
  - Кнопка "Отправить заявку"

**Требование 2.6.5:** Запрос снятия
- Кнопка на каждой карточке вклада: "Отправить на снятие"
- Отправка заявки на одобрение родителю

### 3.4 Масштабируемость
- Структура готова к переходу на PostgreSQL

- Модульная архитектура (отдельные маршруты для auth, deposits, operations, analytics)

### 3.5 Удобство использования

- Интуитивный интерфейс

### 3.6 Тестируемость

- Возможность локального тестирования без интернета

## 4. ТЕХНИЧЕСКИЙ СТЕК

### Backend
- **Runtime:** Node.js 14+ (LTS)
- **Framework:** Express.js 4.18+
- **Database:** SQLite 3 (встроенная, файл `bank.db`)
- **Аутентификация:** 
  - jsonwebtoken (JWT) для авторизации
  - bcryptjs для хеширования паролей
- **Middleware:** cors, body-parser
- **Environment:** dotenv для переменных окружения

### Frontend
- **Library:** React 18
- **Router:** react-router-dom 6+
- **HTTP Client:** axios
- **Icons:** lucide-react
- **Styling:** vanilla CSS (БЕЗ фреймворков типа Tailwind)
- **Storage:** localStorage (для токена и данных)

### Database Schema

**Таблица users**
```
id (INT PK)
username (TEXT UNIQUE)
password (TEXT)
name (TEXT)
role (TEXT: 'admin' | 'child')
parent_id (INT FK to users)
created_at (DATETIME)
```

**Таблица deposits**
```
id (INT PK)
child_id (INT FK)
bank (TEXT: 'mama' | 'papa')
amount (REAL)
current_balance (REAL)
interest_rate (REAL)
period_days (INT)
status (TEXT: 'active' | 'closed' | 'pending')
created_at (DATETIME)
last_interest_calc (DATETIME)
closed_at (DATETIME)
```

**Таблица operations**
```
id (INT PK)
deposit_id (INT FK)
user_id (INT FK)
type (TEXT: 'open' | 'withdraw' | 'interest' | 'penalty')
amount (REAL)
status (TEXT: 'pending' | 'approved' | 'rejected')
notes (TEXT)
requested_at (DATETIME)
approved_at (DATETIME)
approved_by (INT FK)
```

**Таблица interest_log**
```
id (INT PK)
deposit_id (INT FK)
calculated_at (DATETIME)
interest_amount (REAL)
new_balance (REAL)
```

## 5. API ENDPOINTS

### 5.1 Авторизация (/api/auth)
```
POST /api/auth/register
  Body: { username, password, name, role, parent_id? }
  Response: { id, username, name, role }

POST /api/auth/login
  Body: { username, password }
  Response: { token, user: { id, name, role } }
```

### 5.2 Вклады (/api/deposits)
```
GET /api/deposits?child_id=ID
  Response: [ { id, child_id, bank, amount, current_balance, ... } ]

GET /api/deposits/:id
  Response: { ...deposit, calculated_balance, interest_history: [] }
```

### 5.3 Операции (/api/operations)
```
POST /api/operations/request
  Body: { deposit_id, type, amount }
  Response: { id, status: 'pending' }

GET /api/operations/pending
  Auth: admin only
  Response: [ { id, child_id, child_name, type, amount, bank, ... } ]

POST /api/operations/:id/approve
  Auth: admin only
  Response: { status: 'approved', actual_amount?, penalty? }

POST /api/operations/:id/reject
  Auth: admin only
  Response: { status: 'rejected' }
```

### 5.4 Аналитика (/api/analytics)
```
GET /api/analytics/forecast/:depositId
  Response: {
    current_balance,
    in_7_days,
    in_14_days,
    in_30_days,
    in_90_days
  }

GET /api/analytics/stats/:childId
  Response: {
    total_invested,
    total_current,
    deposits_count,
    active_deposits,
    total_interest_earned
  }
```

## 8. ТЕСТИРОВАНИЕ

### 8.1 Функциональное тестирование
- [ ] Регистрация: создание пользователя с двумя ролями
- [ ] Вход: вход с корректными и неправильными данными
- [ ] Создание вклада: Мама и Папа банки с разными суммами
- [ ] Одобрение операции: проверка смены статуса
- [ ] Расчёт процентов: проверка формулы на разных периодах

- [ ] Сумма меньше минимума

- [ ] Невозможно изменить роль через API
