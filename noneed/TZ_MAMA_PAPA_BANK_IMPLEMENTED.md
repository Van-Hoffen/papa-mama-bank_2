# 📋 ТЕХНИЧЕСКОЕ ОПИСАНИЕ И СПЕЦИФИКАЦИЯ (ТЗ РЕАЛИЗАЦИИ)

## Веб-приложение "Мама-Папа Банк"
### Интерактивная образовательная платформа финансовой грамотности для всей семьи

---

## 1. ОБЩЕЕ ОПИСАНИЕ И АРХИТЕКТУРА СИСТЕМЫ

Веб-приложение **"Мама-Папа Банк"** представляет собой полнофункциональную full-stack платформу (Client-Server), разработанную для обучения детей финансовой грамотности, накоплению средств и пониманию работы банковских депозитов под контролем родителей.

### 1.1 Стек технологий
* **Frontend:** Single Page Application (SPA) на базе **React 18 (Vite)**.
  * **Стилизация:** Tailwind CSS для адаптивного интерфейса высокой четкости.
  * **Иконки:** библиотека `lucide-react`.
  * **HTTP-клиент:** Axios с настроенными перехватчиками авторизации.
* **Backend:** RESTful API на базе **Node.js (Express)**.
  * **СУБД:** SQLite3 (`bank.db`) для быстрой и надежной локальной персистентности данных.
  * **Авторизация:** JWT (JSON Web Tokens) с хранением сессии на стороне клиента (`localStorage`).
  * **Безопасность:** Хеширование паролей с помощью `bcryptjs` (10 раундов соли).
  * **Серверный процесс:** Контролируется через `nodemon` в режиме разработки.

## 2. МОДЕЛЬ ДАННЫХ И СХЕМА СУБД

База данных основана на реляционной схеме SQLite со следующими сущностями:

```
                  ┌──────────────┐
                  │    USERS     │
                  └──────┬───────┘
                         │ 1
                         │
                         │ 1..*
                  ┌──────▼───────┐
                  │   DEPOSITS   │◄──────────────┐
                  └──────┬───────┘               │
                         │ 1                     │ 1
                         │                       │
                         │ 1..*                  │ 1..*
                  ┌──────▼───────┐        ┌──────┴───────┐
                  │  OPERATIONS  │        │ INTEREST_LOG │
                  └──────────────┘        └──────────────┘
```

### 2.1 Таблица `users` (Пользователи)
Хранит учетные записи детей и взрослых (включая раздельных банковских администраторов).
* `id` (INTEGER, Primary Key, Auto Increment) — Уникальный идентификатор.
* `username` (TEXT, Unique, Not Null) — Логин пользователя для авторизации.
* `password` (TEXT, Not Null) — Хешированный пароль (bcryptjs).
* `name` (TEXT, Not Null) — Отображаемое имя (например, "Иван", "Мама", "Папа").
* `role` (TEXT, Not Null) — Роль пользователя в системе:
  * `'admin'` — Глобальный супер-администратор (управление пользователями, создание банков).
  * `'bank-admin'` — Администратор конкретного банка (Мама или Папа).
  * `'child'` — Вкладчик (ребёнок).
* `parent_id` (INTEGER, Foreign Key) — Ссылка на родителя (для детей).
* `bank` (TEXT, Nullable) — Привязка администратора к банку (`'mama'`, `'papa'`, `'babushka'` и др.).
* `created_at` (DATETIME, Default: CURRENT_TIMESTAMP) — Дата регистрации.

### 2.2 Таблица `deposits` (Депозиты)
Хранит информацию о созданных детьми вкладах.
* `id` (INTEGER, Primary Key, Auto Increment) — Уникальный идентификатор вклада.
* `child_id` (INTEGER, Foreign Key, Not Null) — ID ребенка, открывшего вклад.
* `bank` (TEXT, Not Null) — Идентификатор банка (`mama`, `papa`, `babushka`).
* `amount` (REAL, Not Null) — Изначальная (стартовая) сумма вклада.
* `current_balance` (REAL, Default 0) — Баланс вклада (при фиксации или закрытии).
* `interest_rate` (REAL, Not Null) — Процентная ставка, действующая по вкладу (например, `0.035` для 3.5%).
* `period_days` (INTEGER, Not Null) — Периодичность начисления процентов в днях (например, `14` или `30`).
* `status` (TEXT, Default `'pending'`) — Текущий статус вклада:
  * `'pending'` — На рассмотрении родителем.
  * `'active'` — Активный вклад (начисляются проценты).
  * `'closed'` — Закрыт (выплачен или досрочно закрыт с учетом штрафов).
* `created_at` (DATETIME, Default: CURRENT_TIMESTAMP) — Дата одобрения/создания вклада.
* `last_interest_calc` (DATETIME, Default: CURRENT_TIMESTAMP) — Дата последнего начисления.
* `closed_at` (DATETIME) — Дата закрытия вклада.
* **Поля динамического изменения условий:**
  * `pending_interest_rate` (REAL) — Предложенная новая процентная ставка.
  * `pending_period_days` (INTEGER) — Предложенный новый период.
  * `pending_penalty_rate` (REAL) — Предложенная новая штрафная ставка.
  * `rate_change_status` (TEXT, Default `'none'`) — Статус согласования новых условий:
    * `'none'` — Изменения отсутствуют.
    * `'pending_child_approval'` — Изменения предложены взрослым, ожидают одобрения ребенком.

### 2.3 Таблица `operations` (Операции)
Хранит все транзакции и запросы на транзакции.
* `id` (INTEGER, Primary Key, Auto Increment) — ID операции.
* `deposit_id` (INTEGER, Foreign Key, Not Null) — Связанный вклад.
* `user_id` (INTEGER, Foreign Key, Not Null) — Инициатор запроса.
* `type` (TEXT, Not Null) — Тип транзакции:
  * `'open'` — Открытие вклада.
  * `'withdraw'` — Снятие средств/закрытие вклада.
  * `'interest'` — Системное начисление процентов.
  * `'penalty'` — Снятие штрафа за досрочное закрытие.
* `amount` (REAL) — Сумма операции.
* `status` (TEXT, Default `'pending'`) — Статус транзакции (`'pending'`, `'approved'`, `'rejected'`).
* `notes` (TEXT) — Комментарии или примечания к операции.
* `requested_at` (DATETIME, Default: CURRENT_TIMESTAMP) — Время подачи заявки.
* `approved_at` (DATETIME) — Время подтверждения операции.
* `approved_by` (INTEGER, Foreign Key) — ID администратора, одобрившего транзакцию.

### 2.4 Таблица `interest_log` (Лог процентов)
Фиксирует исторические начисления процентов по каждому вкладу.
* `id` (INTEGER, Primary Key, Auto Increment) — ID лога.
* `deposit_id` (INTEGER, Foreign Key, Not Null) — Связанный вклад.
* `calculated_at` (DATETIME, Default: CURRENT_TIMESTAMP) — Время начисления.
* `interest_amount` (REAL, Not Null) — Сумма начисленного процента.
* `new_balance` (REAL, Not Null) — Баланс вклада после начисления.

### 2.5 Таблица `bank_settings` (Настройки банковских условий)
Обеспечивает гибкую настройку условий каждого банка.
* `bank` (TEXT, Primary Key) — Уникальный строковый ключ банка (например, `mama`, `papa`, `babushka`).
* `display_name` (TEXT, Not Null) — Понятное для пользователя название (например, "Бабушка-банк").
* `interest_rate` (REAL, Not Null) — Действующая базовая процентная ставка.
* `period_days` (INTEGER, Not Null) — Период начисления процентов в днях.
* `min_amount` (REAL, Not Null) — Минимальная сумма для открытия вклада в этом банке.
* `penalty_rate` (REAL, Not Null) — Процент штрафа за досрочное изъятие средств (например, `0.02` для 2%).

## 3. ФУНКЦИОНАЛЬНОСТЬ И БИЗНЕС-ЛОГИКА СИСТЕМЫ

### 3.1 Разделение прав доступа и Ролевая модель (RBAC)
Система жестко разграничивает видимость данных и возможности администрирования:
1. **Глобальный Администратор (`admin`)**:
   * Видит все банковские условия, все вклады детей и все транзакции.
   * Может создавать новые банки через панель настроек (например, добавить "Бабушка-банк").
   * Имеет право одобрять и отклонять любые операции во всех банках.
2. **Банковский Администратор (`bank-admin` с привязкой к банку `mama`, `papa` и др.)**:
   * Видит только вклады и операции, относящиеся к его банку.
   * Одобряет или отклоняет заявки только в рамках своего банка.
   * Имеет доступ к редактированию условий (процентная ставка, штрафы, минимальный депозит) только своего банка.
3. **Вкладчик (`child`)**:
   * Видит свои открытые вклады, общий баланс и историю операций.
   * Может подавать заявки на открытие новых вкладов в любом доступном банке.
   * Может запрашивать досрочный вывод средств.
   * При изменении родителем процентных ставок видит интерактивные карточки согласования условий.

### 3.2 Динамический расчет сложных процентов
Расчет процентов по вкладам происходит **на лету** при каждом запросе данных. Это предотвращает рассинхронизацию, вызванную пропущенными фоновыми процессами.
* **Формула сложного процента:**  
  $$A = P \times (1 + r)^n$$
  Где:
  * $P$ — начальная сумма депозита (`amount`).
  * $r$ — процентная ставка за период (`interest_rate`).
  * $n$ — количество полностью завершенных периодов с момента открытия вклада:  
    $$n = \lfloor \frac{\text{Количество дней с момента создания}}{\text{period\_days}} \rfloor$$
* При выводе данных баланс рассчитывается автоматически на текущий момент, и пользователь видит точные накопления в реальном времени.

### 3.3 Согласование изменения ставок (Консенсус-протокол)
Если администратор банка решает изменить условия (например, снизить ставку Мама-банка с 3.5% до 3%):
1. Существующие активные вклады детей **не меняют условия насильно**, чтобы сохранить доверие ребенка и обучить его правилам контрактных обязательств.
2. Активные вклады переводятся в статус `rate_change_status = 'pending_child_approval'`, а новые предложенные параметры сохраняются в полях `pending_*`.
3. В кабинете ребенка отображается предупреждающий баннер:  
   *"Взрослые предлагают изменить условия вклада. Старая ставка: X%, Новая ставка: Y%."*
4. Ребенку доступны 2 кнопки выбора:
   * **Одобрить новые условия:** Вклад переводится на новые процентные ставки и периоды.
   * **Отказываюсь (Сохранить старые условия):** Вклад продолжает обслуживаться по ставкам, актуальным на момент его открытия.

### 3.4 Прогнозы доходности (Аналитика)
Система строит динамические прогнозы прироста средств по каждому вкладу на фиксированные временные промежутки:
* **Интервалы прогнозирования:** 7 дней, 14 дней, 30 дней и 90 дней.
* Прогноз рассчитывает будущий сложный процент с учетом текущих или вновь одобренных условий вклада, показывая ребенку финансовую выгоду от долгосрочного сбережения.

### 3.5 Досрочное снятие и Штрафные санкции
* В банках с установленной штрафной ставкой (`penalty_rate` > 0, например, Папа-банк) при запросе досрочного вывода средств система предупреждает ребенка о штрафе.
* При одобрении родителем заявки на досрочное закрытие автоматически создается корректирующая транзакция типа `penalty` на сумму штрафа, которая вычитается из финальной выплаты.

## 4. ИНТЕРФЕЙС И ПОЛЬЗОВАТЕЛЬСКИЙ ОПЫТ (UI/UX)

Приложение спроектировано с акцентом на визуальную эстетику, простоту восприятия детьми и удобство контроля для родителей.

### 4.1 Панель управления Ребенка (`ChildDashboard`)
* **Виджеты общей статистики:** интерактивные карточки с общим балансом, суммой инвестиций, количеством вкладов и общим заработанным процентом.
* **Карусель выбора банков:** ребенок видит карточки доступных банков с их ключевыми условиями (Ставка, Период, Лимиты). Присутствуют индивидуальные иконки и цветовые схемы для каждого банка:
  * 🏠 **Мама-банк** — теплые тона, стабильные накопления.
  * 🏖️ **Папа-банк** — строгие синие тона, высокая доходность на более длинный срок.
  * 👵 **Бабушка-банк** — зеленый цвет, быстрый оборот средств.
* **Интерактивные карточки вкладов:** детальный просмотр каждого вклада с индикатором прогресса текущего периода, накопленным процентом и кнопками управления.
* **Форма отправки запроса на вклад:** открывается в модальном окне, автоматически валидирует минимальные лимиты сумм перед отправкой.

### 4.2 Панель управления Администратора (`AdminDashboard`)
* **Раздел верификации заявок (`Заявки на одобрение`)**: список всех входящих запросов от детей на открытие и закрытие вкладов.
* **Таблица активных вкладов (`Все вклады детей`)**: фильтрация, мониторинг балансов и дат начисления процентов.
* **Раздел настроек банка (`Настройки условий вкладов`)**: форма изменения процентных ставок, периодов, лимитов и штрафов. Условия можно изменять динамически в один клик.
* **Раздел создания банков (`Создание банка`)** — эксклюзивный функционал для Super Admin (`admin`), позволяющий динамически масштабировать систему.

## 5. ИНТЕГРАЦИЯ И ЗАПУСК ПРОЕКТА

### 5.1 API Эндпоинты

#### Авторизация (`/api/auth`)
* `POST /api/auth/register` — регистрация нового пользователя (валидация логина и надежности пароля).
* `POST /api/auth/login` — аутентификация пользователя, генерация JWT.
* `GET /api/auth/me` — проверка текущей сессии по JWT-токену.

#### Депозиты (`/api/deposits`)
* `GET /api/deposits?child_id=ID` — список вкладов (с фильтрацией по банкам для `bank-admin`).
* `POST /api/deposits` — создание заявки на открытие вклада.
* `POST /api/deposits/:id/approve-rate-change` — согласие ребенка с новыми условиями банка.
* `POST /api/deposits/:id/decline-rate-change` — отказ ребенка от новых условий (сохранение старых).

#### Операции/Заявки (`/api/operations`)
* `GET /api/operations/pending` — список ожидающих одобрения заявок (разграничен по банкам для админов).
* `POST /api/operations/request` — создание заявки на снятие денег ребенком.
* `POST /api/operations/:id/approve` — одобрение заявки администратором (смена статуса вклада, применение штрафов).
* `POST /api/operations/:id/reject` — отклонение заявки родителем с указанием причины.

#### Настройки (`/api/settings`)
* `GET /api/settings` — получение актуальных параметров всех банков.
* `POST /api/settings` — добавление нового банка в реестр (Super Admin).
* `PUT /api/settings/:bank` — обновление процентных ставок и лимитов существующего банка.

#### Аналитика (`/api/analytics`)
* `GET /api/analytics/stats/:child_id` — получение сводных финансовых показателей по ребенку.
* `GET /api/analytics/forecast/:deposit_id` — расчет сложного процента на 7, 14, 30, 90 дней вперед.

## ЗАКЛЮЧЕНИЕ И ЦЕННОСТЬ РЕШЕНИЯ
Данная архитектурная реализация платформы **"Мама-Папа Банк"** полностью закрывает требования по разделению зон ответственности родителей, предоставляет гибкую систему динамической настройки банковских продуктов без изменения программного кода и обучает детей договорным отношениям через механизм согласования процентных ставок. Приложение готово к масштабированию и полноценному семейному использованию.
