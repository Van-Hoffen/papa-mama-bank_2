**«Мама-Папа Банк»** — это полнофункциональное семейное веб-приложение для обучения детей финансовой грамотности, навыкам сбережения и пониманию работы банковских депозитов через систему виртуальных вкладов под мягким контролем родителей.

---

## 🌟 Основные ключевые моменты проекта

### 1. 🏦 Многобанковая система с гибкими условиями
- **Разные банковские продукты:** В системе предустановлены **Мама-Банк** (короткие сроки, гибкое пополнение) и **Папа-Банк** (высокая процентная ставка, длинный срок, штраф за досрочный вывод).
- **Динамическое создание банков:** Супер-администратор может добавлять новые банки (например, *«Бабушка-Банк»*) с собственными уникальными параметрами: процентной ставкой, периодом начисления, минимальным депозитом и штрафной ставкой.

### 2. 🔐 Ролевая модель доступа (RBAC) и разделение банков
- **Супер-администратор (`admin`):** Имеет полный доступ ко всем банкам, глобальным настройкам, созданию новых банков и управлению пользователями.
- **Банковский администратор (`bank-admin`):** Управляет только своим банком (Мама-Админ видит и одобряет только операции в Мама-Банке, Папа-Админ — в Папа-Банке).
- **Вкладчик / Ребёнок (`child`):** Открывает вклады, запрашивает пополнение и вывод средств, видит финансовые прогнозы и согласовывает изменения ставок.

### 3. 📈 Автоматическое начисление сложных процентов
- Проценты рассчитываются по формуле сложного процента с капитализацией за каждый завершенный период:
  $$A = P \times (1 + r)^n$$
- Расчёт выполняется «на лету» при каждом запросе, обеспечивая точность до копейки без риска сбоев фоновых демонов.

### 4. 🤝 Консенсус-протокол изменения процентных ставок
- Если родитель меняет условия банка (например, снижает процентную ставку), действующие вклады детей **не меняются автоматически**.
- Ребёнку отправляется запрос на согласование новых условий. Ребёнок может:
  - **Принять новые условия:** Вклад переводится на новую процентную ставку.
  - **Отклонить:** Вклад продолжает обслуживаться по первоначальной ставке, на которую ребёнок соглашался при открытии.

### 5. 💳 Пополнение вкладов и досрочное изъятие
- **Пополнение вкладов:** Ребёнок может подать заявку на пополнение существующего активного вклада.
- **Защита от досрочного снятия:** При закрытии вклада до истечения срока система автоматически рассчитывает штраф (в соответствии с правилами банка) и запрашивает подтверждение.

### 6. 🛡️ Полный родительский контроль и система заявок
- Ни одна денежная операция (открытие вклада, пополнение, вывод средств) не происходит без одобрения родителя.
- В панели администратора отображаются удобные карточки заявок с возможностью одобрения или отклонения с комментарием.

### 7. 📊 Финансовая аналитика и визуальные прогнозы
- **Прогнозирование дохода:** Наглядный расчёт будущей прибыли по вкладу на **7, 14, 30 и 90 дней**.
- **Интерактивный UI:** Прогресс-бары текущего периода начисления, графическая карусель банков, финансовые индикаторы и детальная история всех операций.

## 🚀 Быстрый запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/Van-Hoffen/papa-mama-bank_2.git
cd papa-mama-bank_2
```

### 2. Запуск Backend
```bash
cd backend
npm install
npm start
```
*Backend запустится на `http://localhost:5000` (SQLite БД инициализируется автоматически при первом запуске).*

### 3. Запуск Frontend
```bash
cd frontend
npm install
npm run dev
```
*Frontend будет доступен по адресу `http://localhost:3000` (или `http://localhost:5173`).*

## 📡 Основные API Endpoints

- **Авторизация:** `POST /api/auth/login`, `POST /api/auth/register`, `GET /api/auth/me`
- **Депозиты:** `GET /api/deposits`, `POST /api/deposits`, `POST /api/deposits/:id/topup`
- **Согласование ставок:** `POST /api/deposits/:id/approve-rate-change`, `POST /api/deposits/:id/decline-rate-change`
- **Заявки и операции:** `GET /api/operations/pending`, `POST /api/operations/:id/approve`, `POST /api/operations/:id/reject`
- **Управление банками:** `GET /api/settings`, `POST /api/settings`, `PUT /api/settings/:bank`
- **Аналитика:** `GET /api/analytics/stats/:child_id`, `GET /api/analytics/forecast/:deposit_id`
