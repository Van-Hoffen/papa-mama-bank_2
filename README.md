# Мама-Папа Банк - Образовательная платформа для детей

## Описание проекта

**Название:** Мама-Папа Банк  
**Тип:** Веб-приложение (SPA + Backend API)  
**Назначение:** Образовательная платформа для обучения детей финансовой грамотности через систему виртуальных вкладов с разными условиями.

**Целевая аудитория:**
- Дети (7-18 лет) - пользователи-вкладчики
- Родители (25+ лет) - администраторы системы

**Основная идея:** 
Система состоит из двух "банков" (Мама-банк и Папа-банк) с разными условиями хранения денег. Ребёнок выбирает банк, открывает вклад, и система автоматически начисляет проценты. Все операции требуют одобрения от родителя через систему заявок.

## Установка и запуск

### Требования
- Node.js 14+
- npm

### Установка backend

```bash
cd backend
npm install
npm start
```

### Установка frontend

```bash
cd frontend
npm install
npm run dev
```

### Переменные окружения

#### Backend (.env)
```
PORT=5000
JWT_SECRET=your-super-secret-key
DATABASE=./bank.db
NODE_ENV=development
```

#### Frontend (.env)
```
VITE_REACT_APP_API_URL=http://localhost:5000/api
```

## Особенности

- **Двухбанковая модель**: Мама-банк (3.5% за 14 дней) и Папа-банк (11% за 30 дней)
- **Система одобрения операций**: Все операции требуют подтверждения от родителя
- **Автоматическое начисление процентов**: Сложный процент с капитализацией
- **Разделение доступа**: Мама-администратор видит только Мама-банк, Папа-администратор - только Папа-банк
- **Адаптивный дизайн**: Работает на мобильных устройствах и десктопах

## Технический стек

### Backend
- **Runtime:** Node.js 14+ (LTS)
- **Framework:** Express.js 4.18+
- **Database:** SQLite 3 (встроенная, файл `bank.db`)
- **Аутентификация:** jsonwebtoken (JWT) для авторизации, bcryptjs для хеширования паролей
- **Middleware:** cors, body-parser
- **Environment:** dotenv для переменных окружения

### Frontend
- **Library:** React 18
- **Router:** react-router-dom 6+
- **HTTP Client:** axios
- **Icons:** lucide-react
- **Styling:** vanilla CSS

## API Endpoints

### Авторизация
- `POST /api/auth/register` - регистрация пользователя
- `POST /api/auth/login` - вход в систему
- `GET /api/auth/current-user` - получить текущего пользователя

### Вклады
- `GET /api/deposits?child_id=ID` - получить вклады ребёнка
- `GET /api/deposits/:id` - получить конкретный вклад
- `POST /api/deposits` - создать новый вклад

### Операции
- `POST /api/operations/request` - запросить операцию
- `GET /api/operations/pending` - получить ожидающие операции (для администраторов)
- `POST /api/operations/:id/approve` - одобрить операцию
- `POST /api/operations/:id/reject` - отклонить операцию

### Аналитика
- `GET /api/analytics/forecast/:depositId` - прогноз доходности вклада
- `GET /api/analytics/stats/:childId` - статистика по ребёнку
- `GET /api/analytics/overall-stats` - общая статистика (для администраторов)
